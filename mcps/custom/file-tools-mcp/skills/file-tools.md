## File Tools

Use the file-tools MCP for all document and image operations:

- **`read_document`** — Extract text and data from PDF, DOCX, XLSX, PPTX files. For XLSX: supports range-based reading (start_cell/end_cell), formula view (show_formulas), and metadata (validations, merged cells, tables). Use this instead of pip-installing libraries manually.
- **`write_docx`** — Create or modify Word documents. Full document support: paragraphs (mixed formatting via runs array, spacing, indentation), headings, lists (nested up to 5 levels), tables (merge cells, add/delete rows, column widths, styling), images (with alignment), headers & footers, page numbers ({page} of {total}), hyperlinks, table of contents (auto-renders in Collabora), search & replace (across body, tables, headers), section breaks, page setup.
- **`write_xlsx`** — Create or modify Excel workbooks. Full spreadsheet support: sheet management (create/delete/rename/copy/protect), cell operations (write/formula/merge/unmerge/copy range with styles/clear range), row/column operations (insert/delete rows & columns, set width/height, auto-fit), formatting (bold/italic/underline/strikethrough/fill/border/number format/alignment/wrap text/protection), features (native Excel tables with styles, data validation & dropdowns, conditional formatting — color scale/data bar/icon set/cell_is/formula, charts — bar/line/pie/scatter/area with axis titles, auto-filter, freeze panes, images, named ranges).
- **`write_pptx`** — Create or modify PowerPoint presentations. Full presentation support: slides (layouts, duplicate, reorder), shapes (25+ auto-shapes including flowchart, arrows, callouts — with text, fill, line), charts (column/bar/line/pie/doughnut/scatter/area/radar), tables (cell formatting, merge), speaker notes, transitions (fade/push/wipe/dissolve), connectors (lines/arrows between shapes), hyperlinks, bullet points with levels, freeform shapes, slide numbers, presentation properties.
- **`write_pdf`** — Create PDF from HTML or Markdown content with custom CSS, page size, and margins.
- **`edit_pdf`** — Edit existing PDFs. Operations: merge (combine PDFs), split (extract pages), rotate, delete/reorder pages, add watermark (text, diagonal, opacity), add text/images at positions, replace_text (edit EXISTING text in place — {find, replace}, matches the original size/color/family, replace:"" deletes), redact (permanently remove text or a region — privacy-grade, content is gone from the file), add annotations (highlight, underline, strikeout, notes), encrypt (AES-256 with passwords and permissions), decrypt, compress (with size reduction report), set metadata, extract embedded images, OCR scanned pages (makes searchable via tesseract).
- **`screenshot_document`** — Render document pages as images to visually inspect layout. Supports PDF, DOCX, XLSX, PPTX. Pages shown inline so you can see exactly how the document looks. Use after creating or modifying a document to verify visual appearance. Parameters: path, pages (default "1", supports "1-3", "-1" for last, "all"), sheet (XLSX only — sheet name or index), dpi (default 150).
- **`pdf_to_images`** — Render PDF pages as PNG or JPEG files at configurable DPI. First page shown inline. Use for creating persistent image assets from PDFs (screenshot_document is better for quick visual review).
- **`images_to_pdf`** — Combine one or more images into a PDF. Supports page sizes (A4, Letter, original) and fit modes (contain, cover, stretch).
- **`convert_document`** — Convert between formats: DOCX→PDF, XLSX→PDF, PPTX→PDF, CSV→XLSX, MD→PDF, Image→PDF, PDF→PNG, etc.
- **`edit_image`** — Pipeline of image operations: resize, crop, rotate, flip, convert_format (png/jpeg/webp), compress (quality control). Lightroom-style tone: exposure, highlights, shadows, whites, blacks, clarity, dehaze. Color: saturation, color_temperature, split_tone, hsl_adjust, grayscale, sepia, invert. Detail: sharpness, denoise, blur, sharpen, edge_enhance, emboss. Effects: add_text, add_border, paste_image, add_watermark, remove_background (AI-powered — outputs transparent PNG or custom background color), blur_region (blur rectangular area for privacy — faces, license plates, addresses).
- **`analyze_image`** — Full image analysis: file info (format, dimensions, size, color mode), EXIF data (camera, date, exposure, ISO, focal length, GPS coordinates), luminance histogram, zone balance, color channels, dynamic range, temperature, suggested fixes.
- **`create_chart`** — Create a chart IMAGE (PNG): shown inline in the chat, optionally saved as a file for embedding in documents (docx/pdf/xlsx have no live charts). Types: bar, column, line, pie, doughnut, scatter, area, heatmap, histogram. Styles: modern, dark, minimal, presentation. Routing: prefer `display_ui` (display MCP) for purely in-chat visualization when it is available — use `create_chart` for chart FILES to embed in documents, and as the in-chat chart path where `display_ui` isn't offered.
- **`preview_document`** — Show a live Collabora preview in the dashboard. Write tools trigger this automatically after saving — use it manually to preview files the user uploaded or that you downloaded.

### Key Behaviors

- **Auto-preview**: All write/edit tools automatically show results to the user at the end of each turn. Documents (DOCX, XLSX, PPTX, PDF) get a live Collabora preview (view, scroll, zoom, fullscreen, download). Edited images get an inline preview with full-resolution download link. You do NOT need to call `display_images`, `preview_document`, or `send_file` after these tools — previews and downloads are provided automatically.
- Even if you make multiple edits to the same file in one turn, only the final version is previewed (no intermediate previews clutter the chat).
- Images from the user's workspace can be embedded in documents by passing their file paths to the write tools.
- When creating files, always save them to the user's workspace directory (e.g., `users/{username}/workspace/report.docx`).
- Use `create_new: true` on write_docx/write_xlsx/write_pptx when you want to start a fresh document from scratch instead of editing the existing file.
- **Visual review workflow**: When creating or editing documents (especially PPTX presentations and complex layouts), work step-by-step. After creating each slide or page, use `screenshot_document` to see the visual result (only you see these — not the user). Check for: image overflow, text fitting, table sizing, layout balance. Use `set_shape_format` (PPTX) to resize/reposition elements if needed. Screenshot again to confirm before moving to the next section.
